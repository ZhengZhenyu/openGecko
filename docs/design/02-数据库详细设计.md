## 4. 索引策略

### 4.1 必要索引原则

1. **所有外键列**：所有 FK 列都应有索引（防止全表扫描关联查询）
2. **过滤高频列**：`community_id`、`status`、`work_status`、`is_read`、`created_at`
3. **排序列**：`scheduled_at`、`stat_date`、`snapshot_at`
4. **复合索引**：`(community_id, period_type, period_start)` 用于聚合统计；`(scheduled_at, status)` 用于会议提醒轮询

### 4.2 SQLite 与 PostgreSQL 兼容说明

- SQLite 不支持直接 `ALTER COLUMN`，Alembic 使用 **batch mode** 处理
- 时间戳统一使用不带时区的 `DATETIME`，应用层处理时区
- Boolean 字段在 SQLite 中存储为 INTEGER（0/1），SQLAlchemy 自动处理
