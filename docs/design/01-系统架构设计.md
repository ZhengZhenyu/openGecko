# openGecko - 系统架构设计文档

**项目名称**: openGecko - 多社区运营管理平台
**文档版本**: v5.0
**更新日期**: 2026-03-02
**架构风格**: 前后端分离 + 多租户 SaaS + 分层六边形架构
**品牌定位**: Manage All, Publish Everywhere

## 0. 架构决策记录（ADR）

| # | 决策 | 选型 | 主要理由 |
|---|------|------|----------|
| ADR-001 | Web 框架 | FastAPI + Uvicorn | 原生异步、自动 OpenAPI、基于类型注解的依赖注入，生产性能优于 Django REST |
| ADR-002 | ORM | SQLAlchemy 2.0 | 声明式映射、类型安全、支持异步引擎，Alembic 自动迁移 |
| ADR-003 | 前端框架 | Vue 3 + Composition API | `<script setup>` 模式下类型推断完整，Pinia 比 Vuex 更轻量 |
| ADR-004 | 多租户策略 | 共享数据库 + 行级隔离 | `community_id` 列过滤，部署成本低，租户规模可控 |
| ADR-005 | 认证方案 | JWT（无状态）+ bcrypt | 去除服务端 session 状态，水平扩展无需 sticky session |
| ADR-006 | 密钥管理 | Fernet（对称）+ 环境变量 | 对称加密满足凭证存储需求；密钥注入符合 12-Factor App 原则 |
| ADR-007 | 数据库选型 | SQLite（dev）/ PostgreSQL（prod）| 开发零配置；生产使用 pg 以获得并发写、JSONB、全文索引 |
| ADR-008 | 静态文件服务 | Nginx | 将静态资源从应用进程卸载，支持 gzip、缓存头和 range 请求 |

### 1.3 核心设计原则

| 原则 | 实现方式 |
|------|----------|
| **关注点分离** | Router → Service → Repository 三层，路由层不含业务逻辑 |
| **依赖倒置** | 业务层通过抽象接口调用外部适配器，便于单元测试 Mock |
| **租户隔离** | 所有社区数据查询强制附加 `community_id` 过滤，中间件层验证 Header |
| **幂等性** | 发布操作携带 `idempotency_key`，防止网络重试导致重复发布 |
| **失败隔离** | 外部集成（WeChat/Email）异常不抛出 500，降级返回提示 |
| **最小权限** | RBAC 三级（superuser / admin / user），API 维度权限检查 |

### 3.2 Pinia Store 设计

| Store | 状态 | 说明 |
|-------|------|------|
| `authStore` | `token`, `user`, `isLoggedIn` | JWT Token 存 localStorage，自动附加到 Axios |
| `communityStore` | `currentCommunity`, `communities`, `communityId` | 当前社区，切换时触发 `X-Community-Id` 更新 |
| `userStore` | `profile` | 当前用户详情（含角色） |
| `featureStore` | `enableInsightsModule` | 功能开关，从后端拉取 |

### 3.3 Axios 拦截器

```typescript
// 请求拦截：自动附加认证和社区信息
axios.interceptors.request.use(config => {
  const token = authStore.token
  if (token) config.headers.Authorization = `Bearer ${token}`

  const communityId = communityStore.communityId
  if (communityId) config.headers['X-Community-Id'] = communityId

  return config
})

// 响应拦截：统一错误处理
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) authStore.logout()
    return Promise.reject(error)
  }
)
```

## 4. 安全架构

参考 OWASP Top 10（2021）和 NIST SP 800-63B 身份认证标准进行设计。

### 4.1 威胁边界与防御层

```
┌─────────────────────────────────────────┐
│  Internet                                │
│         ↓ HTTPS (TLS 1.2+)              │
│  ┌──────────────────────────────────┐   │
│  │ Nginx Layer                      │   │
│  │  • Rate Limiting (10 req/s/IP)   │   │  ← DDoS / Brute Force
│  │  • HSTS / Secure Headers         │   │  ← Protocol Downgrade
│  │  • 隐藏服务器版本信息 (server_tokens off) │   │
│  └──────────────────────────────────┘   │
│         ↓ HTTP                          │
│  ┌──────────────────────────────────┐   │
│  │ FastAPI Middleware Layer         │   │
│  │  • CORS Whitelist                │   │  ← CSRF / Origin 劫持
│  │  • TrustedHost Middleware        │   │  ← Host Header 注入
│  │  • RequestID (tracing)           │   │
│  │  • JWT 验证 + 过期检测           │   │  ← 认证绕过
│  │  • Community Isolation 校验      │   │  ← 越权访问
│  └──────────────────────────────────┘   │
│         ↓                               │
│  ┌──────────────────────────────────┐   │
│  │ Business Logic Layer             │   │
│  │  • Pydantic Schema 输入验证      │   │  ← 注入攻击
│  │  • RBAC 权限检查                 │   │  ← 水平越权
│  │  • SQLAlchemy ORM（禁止裸 SQL）  │   │  ← SQL 注入
│  │  • 文件类型白名单验证            │   │  ← 文件上传攻击
│  └──────────────────────────────────┘   │
│         ↓                               │
│  ┌──────────────────────────────────┐   │
│  │ Data Layer                       │   │
│  │  • Fernet 加密渠道凭证           │   │  ← 数据泄露
│  │  • bcrypt (cost=12) 密码哈希     │   │  ← 彩虹表攻击
│  │  • 敏感字段审计日志              │   │  ← 不可否认性
│  └──────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

### 4.2 认证与授权流程

```mermaid
sequenceDiagram
    participant C as Client (Browser)
    participant N as Nginx
    participant API as FastAPI
    participant DB as PostgreSQL

    Note over C,N: 登录流程
    C->>N: POST /api/auth/token {username, password}
    N->>API: proxy_pass (rate limited: 5 req/min)
    API->>DB: SELECT user WHERE username=? (parameterized)
    DB-->>API: User{hashed_password, is_active}
    API->>API: bcrypt.checkpw(password, hashed_password)
    alt 验证失败（密码错或账号禁用）
        API-->>C: 401 Unauthorized（不区分原因，防止用户枚举）
    else 验证成功
        API->>API: jwt.encode({sub, exp: now+8h, iat})
        API-->>C: {access_token, token_type: "bearer", expires_in: 28800}
    end

    Note over C,API: 授权请求流程
    C->>API: GET /api/contents<br/>Authorization: Bearer <token><br/>X-Community-Id: 42
    API->>API: jwt.decode(token, SECRET, algorithms=["HS256"])
    API->>API: 校验 exp / iat / sub 字段
    API->>DB: SELECT user WHERE id=sub AND is_active=true
    API->>DB: SELECT community_user WHERE user_id=? AND community_id=42
    alt community_id 不匹配或无成员记录
        API-->>C: 403 Forbidden
    else 授权通过
        API->>DB: SELECT contents WHERE community_id=42 AND ...
        API-->>C: 200 [{content}]
    end
```

```python
# backend/app/core/dependencies.py

# L1: 令牌解码 + 用户激活态校验
async def get_current_user(token: str = Depends(oauth2_scheme), db: Session = Depends(get_db)) -> User:
    try:
        payload = jwt.decode(token, settings.SECRET_KEY, algorithms=["HS256"])
        user_id: int = payload.get("sub")
    except JWTError:
        raise HTTPException(status_code=401, detail="无效的认证令牌")
    user = db.get(User, user_id)
    if not user or not user.is_active:
        raise HTTPException(status_code=401, detail="用户不存在或已禁用")
    return user

# L2: 社区隔离（返回 int，非对象，避免额外 DB 查询）
def get_current_community(x_community_id: int = Header(..., alias="X-Community-Id")) -> int:
    return x_community_id

# L3: 社区管理员校验
async def get_community_admin(
    user: User = Depends(get_current_active_user),
    community_id: int = Depends(get_current_community),
    db: Session = Depends(get_db),
) -> User:
    membership = db.query(CommunityUser).filter_by(
        user_id=user.id, community_id=community_id
    ).first()
    if not membership or membership.role not in ("admin", "superuser"):
        raise HTTPException(status_code=403, detail="需要社区管理员权限")
    return user

# L4: 内容级权限（owner / collaborator / admin / superuser 均可编辑）
async def check_content_edit_permission(content_id: int, ...) -> Content:
    content = db.query(Content).filter_by(id=content_id, community_id=community_id).first()
    if not content:
        raise HTTPException(status_code=404)
    allowed = (
        content.owner_id == user.id
        or user.id in [c.user_id for c in content.collaborators]
        or is_community_admin(user, community_id, db)
    )
    if not allowed:
        raise HTTPException(status_code=403)
    return content
```

### 4.4 渠道凭证加密（Fernet）

```python
# backend/app/core/security.py

import base64, hashlib
from cryptography.fernet import Fernet

def _derive_fernet_key(secret: str) -> bytes:
    """从 JWT_SECRET_KEY 派生 32 字节 URL-safe base64 key（确定性派生，无需额外存储 key）"""
    digest = hashlib.sha256(secret.encode()).digest()
    return base64.urlsafe_b64encode(digest)

def encrypt_credential(value: str, secret: str) -> str:
    return Fernet(_derive_fernet_key(secret)).encrypt(value.encode()).decode()

def decrypt_credential(token: str, secret: str) -> str:
    return Fernet(_derive_fernet_key(secret)).decrypt(token.encode()).decode()

def mask_credential(value: str) -> str:
    """API 响应脱敏：显示末 4 位，防止凭证全量暴露"""
    return "****" + value[-4:] if len(value) > 4 else "****"
```

```python
# backend/app/core/logging.py
import logging, json, time
from contextvars import ContextVar

request_id_ctx: ContextVar[str] = ContextVar("request_id", default="-")
community_id_ctx: ContextVar[int] = ContextVar("community_id", default=0)

class JSONFormatter(logging.Formatter):
    def format(self, record: logging.LogRecord) -> str:
        return json.dumps({
            "ts":           time.strftime("%Y-%m-%dT%H:%M:%SZ", time.gmtime()),
            "level":        record.levelname,
            "logger":       record.name,
            "msg":          record.getMessage(),
            "request_id":   request_id_ctx.get(),
            "community_id": community_id_ctx.get(),
            "module":       record.module,
            "line":         record.lineno,
            **(record.__dict__.get("extra", {})),
        }, ensure_ascii=False)

# 示例日志输出
```

### 5.4 请求追踪（Request ID）

```python
# backend/app/middleware/request_id.py
import uuid
from starlette.middleware.base import BaseHTTPMiddleware

class RequestIDMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        req_id = request.headers.get("X-Request-ID") or uuid.uuid4().hex
        request_id_ctx.set(req_id)
        response = await call_next(request)
        response.headers["X-Request-ID"] = req_id   # 透传回客户端
        return response
```

### 7.2 软删除策略

关键业务表（`users`、`contents`、`communities`）使用 `deleted_at` 软删除，保留数据用于审计和恢复：

```python
# SQLAlchemy Mixin
class SoftDeleteMixin:
    deleted_at: Mapped[datetime | None] = mapped_column(nullable=True, index=True)

    @classmethod
    def active(cls, query):
        """过滤器：仅返回未删除记录"""
        return query.filter(cls.deleted_at.is_(None))
```

### 7.3 时间戳规范

| 字段 | 类型 | 说明 |
|------|------|------|
| `created_at` | `TIMESTAMPTZ` | 创建时间，由数据库 `DEFAULT NOW()` 填充 |
| `updated_at` | `TIMESTAMPTZ` | 更新时间，SQLAlchemy `onupdate` 自动维护 |
| `deleted_at` | `TIMESTAMPTZ NULL` | 软删除时间戳 |
| `published_at` | `TIMESTAMPTZ NULL` | 业务发布时间（与 created_at 区分）|

所有时间戳统一使用 UTC（`TIMESTAMPTZ`），前端负责转换为本地时区显示。

```python
# 微信 access_token 本地缓存（避免每次请求都调外部 API）
class WeChatTokenCache:
    _token:      str | None = None
    _expires_at: float      = 0

    def get_token(self, appid: str, secret: str) -> str:
        if time.time() < self._expires_at - 60:   # 提前 60s 刷新
            return self._token
```

### 11.4 文件上传优化

```python
# 流式读取 + 分块写盘，避免大文件 OOM
async def save_upload(file: UploadFile, dest: Path) -> int:
    total = 0
    with dest.open("wb") as f:
        while chunk := await file.read(64 * 1024):  # 64KB chunks
            if total + len(chunk) > MAX_FILE_SIZE:
                dest.unlink(missing_ok=True)
                raise HTTPException(413, "文件超过限制")
            f.write(chunk)
            total += len(chunk)
    return total
```

- 无显式缓存层，数据库成为唯一热点
