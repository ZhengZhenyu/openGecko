# openGecko - 产品路线图

**更新日期**: 2026-03-02
**文档说明**: 本文档记录已交付里程碑和规划中的未来功能

---

## 已交付里程碑

### Phase 1 — 基础认证、RBAC 与多租户 ✅

- JWT 认证 + 首次启动初始化流程（临时账号自动删除机制）
- 三级 RBAC（superuser / admin / user）
- 多租户数据隔离（community_id 强过滤 + X-Community-Id Header）
- 社区管理 CRUD + 成员管理
- 用户管理（全局，superuser 专属）
- 审计日志系统
- 密码重置（SMTP 邮件 + 一次性 Token）

### Phase 2 — 内容管理与多渠道发布 ✅

- DOCX / Markdown 文件上传与格式转换（图片自动提取）
- 在线 Markdown 编辑器（md-editor-v3）+ 实时预览
- 内容工作流（draft → reviewing → approved → published）
- 内容所有权模型（created_by / owner / collaborators / assignees）
- 微信公众号 API 发布（草稿创建 + 图片上传）
- Hugo / CSDN / 知乎发布
- 发布记录追踪与渠道配置（Fernet 加密凭证）
- 内容日历排期视图（FullCalendar + 拖拽）

### Phase 3 — 社区治理 ✅

- 委员会管理（成员信息、角色、任期）
- 会议管理（状态追踪、议程纪要、附件、ICS 导出）
- 会议日历视图
- 个人工作台（跨社区内容 + 会议任务汇总）
- 社区总览仪表板（内容统计、成员数、近期会议、发布趋势）
- 工作量总览（superuser 视角，跨社区对比）

### Phase 4 — 活动、人脉与生态洞察 ✅

- 活动管理全生命周期（CRUD、议程、人员分工、签到、反馈）
- SOP 模板库（可复用 Checklist，支持强制项/负责角色/截止偏移）
- 活动反馈关联 GitHub Issue（IssueLink 追踪）
- 人脉档案（PersonProfile 统一人脉数据库）
- 社区角色记录（CommunityRole，支持多社区多任期）
- Campaign 管理（触达活动、联系人、接触记录）
- 生态项目管理（全字段编辑、数据快照、贡献者管理）
- 生态洞察仪表板（贡献者影响力、企业分布、趋势）
- 素材库（图片/文档素材管理，编辑器内一键插入）
- 设计任务（内容关联设计需求，指定设计师，追踪进度）
- 消息通知系统（站内通知，未读徽标）
- 微信数据统计（文章数据录入与趋势分析）
- Alembic 迁移规范化（18个历史迁移合并为单一 001_initial_schema）

---

## 规划中功能

### Phase 5 — 高级工作流与效率工具 📋

- 内容模板库（快速创建标准化内容结构）
- 批量操作（批量审批、批量发布、批量状态更新）
- Webhook / 外部集成（发布成功后触发回调）
- Kanban 看板视图（拖拽改变内容/任务状态）

### Phase 6 — 生产级加固 📋

- PostgreSQL 迁移指南与最佳实践
- 监控告警接入（Prometheus + Grafana）
- 审计日志查询 UI 页面
- HTTPS / Let's Encrypt 配置向导
- 健康检查端点增强

---

## 暂不计划实现

- 移动端适配（桌面端优先）
- OAuth / SSO 第三方登录
- 实时协同编辑（WebSocket）
- 内容评论与 @ 提醒
- 微信数据自动同步（需要认证资质）
